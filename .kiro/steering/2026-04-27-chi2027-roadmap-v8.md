## §2. AMT Technical Specification

### 2.1 Operator catalog (24 independent operators)

All operators extracted from existing `patches/*.js` code. No new engineering
required for basic operator separation — each is already a standalone block
in the IIFE.

#### 2.1.1 Low family (L1–L13) — aggressive degradation

| ID | Operator | WCAG | Prior layer hypothesis |
|---|---|---|---|
| L1 | Semantic landmark → `<div>` (nav/main/header/footer/article/section/aside) | SC 1.3.1 | semantic + possible visual side-effect |
| L2 | Remove all `aria-*` + `role` globally | SC 2.5.3 | pure semantic |
| L3 | Remove all `<label>` elements | F68 / SC 4.1.2 | semantic + functional (form control identification) |
| L4 | Remove keyboard event handlers (onkeydown/up/press) | SC 2.1.1 | pure functional |
| L5 | Wrap interactives in **closed Shadow DOM** | novel (no WCAG equiv) | structural + semantic |
| L6 | `h1–h6` → `<div>` with CSS font-size preserved | F2 / SC 1.3.1 | pure semantic (visual preserved) |
| L7 | Remove `alt` + `aria-label` + `title` from images | F65 / SC 1.1.1 | pure semantic |
| L8 | Remove `tabindex` attribute | F44 / SC 2.4.3 | functional-dominant |
| L9 | `thead/tbody/tfoot/th` → `div/td` | F91 / SC 1.3.1 | semantic + possible visual |
| L10 | Remove `<html lang>` attribute | SC 3.1.1 | pure semantic (weak effect) |
| L11 | `<a href>` → `<span onclick>` (keep blue + underline) | F42 / SC 1.3.1, 2.1.1 | semantic only (visual + functional preserved) |
| L12 | Duplicate IDs on adjacent elements (≤5) | F77 / SC 4.1.1 | semantic (breaks aria references) |
| L13 | `onfocus="this.blur()"` keyboard trap | F55 / SC 2.1.1, 3.2.1 | pure functional |

#### 2.1.2 Midlow family (ML1–ML3) — pseudo-compliance edge cases

| ID | Operator | WCAG | Prior layer hypothesis |
|---|---|---|---|
| ML1 | `<button>` with no text content → `<div>` (icon-only button) | SC 4.1.2 | semantic (conditional, sparse) |
| ML2 | `role="button"` elements: remove keydown/up + cloneReplace to strip JS-registered listeners | SC 2.1.1 | pure functional (novel clone-replace) |
| ML3 | `<input>` without placeholder: remove `<label for>` + aria-label/aria-labelledby on input | F68 / SC 4.1.2 | semantic (conditional, targeted) |

**Midlow design signal**: Pseudo-compliance = "ARIA written but handlers
broken" — the **most common real-world failure mode** per WebAIM 2024
(96.3% of front pages have detectable a11y errors, most being subtle
rather than total).

#### 2.1.3 High family (H1–H8) — positive enhancement (novel direction)

| ID | Operator | WCAG | Prior layer hypothesis |
|---|---|---|---|
| H1 | Auto-generate `aria-label` for interactive elements lacking accessible names | SC 4.1.2 | semantic positive |
| H2 | Insert skip-nav link at **body end** + `#main-content` anchor | SC 2.4.1 | semantic + navigation, zero visual disruption |
| H3 | Associate labels with form controls (visually-hidden `<label>` or aria-label fallback) | SC 4.1.2 | semantic positive |
| H4 | Add landmark `role` to header/nav/main/footer/aside/form | SC 1.3.1 | semantic positive |
| H5a | Auto-generate `alt` text for images (reverse-engineer from filename) | SC 1.1.1 | semantic positive |
| H5b | Add `lang="en"` to `<html>` | SC 3.1.1 | semantic positive (weak) |
| H5c | Auto-generate `aria-label` for empty-text `<a>` (from href) | SC 2.4.4 | semantic positive |
| H6 | Add `aria-required="true"` for required form inputs | SC 3.3.2 | semantic (state) |
| H7 | Add `aria-current="page"` for nav links matching current URL | SC 2.4.8 | semantic (state) |
| H8 | Add `scope="col/row"` to table header cells | SC 1.3.1 | semantic positive |

**High is a novel direction in a11y agent research**. Existing work
measures agent robustness to perturbation; we measure whether **fixing**
a11y helps agents. This asymmetric framing — environment-to-agent vs
agent-robustness — is a non-trivial contribution.

### 2.2 Operator groupings: additional analysis dimensions

- **13 Low** operators span 6 WCAG success criteria categories
- **3 Midlow** operators target conditional edge cases
- **8 High** operators span 5 WCAG positive-conformance categories
- **Total WCAG coverage**: 11 distinct Success Criteria (SC 1.1.1, 1.3.1, 2.1.1, 2.4.1, 2.4.3, 2.4.4, 2.4.8, 2.5.3, 3.1.1, 3.3.2, 4.1.1, 4.1.2) across A and AA levels — wider than any prior work on web agent a11y

### 2.4 DOM-level signature (12 dimensions)

Each operator is measured against a **12-dimensional DOM audit vector**
*before vs after* apply, averaged across 13 task URLs × 3 independent
page loads to reduce noise.

| Category | Dim | Measurement | Instrument |
|---|---|---|---|
| **DOM** | D1 | tag counts changed | HTML diff |
| | D2 | attribute set diff (added/removed/modified) | attribute enumeration |
| | D3 | node count delta (added/removed) | node counter |
| **A11y tree** | A1 | role assignments changed | `accessibility.snapshot()` diff |
| | A2 | accessible name changed count | a11y tree serialization diff |
| | A3 | ARIA state/property changes (expanded, checked, disabled, etc.) | property diff |
| **Visual** | V1 | SSIM on full-page screenshot | `scikit-image.ssim` |
| | V2 | bounding box shift for key interactives (max in px) | `getBoundingClientRect` diff |
| | V3 | WCAG contrast ratio delta for text elements | contrast audit |
| **Functional** | F1 | interactive element count delta | `querySelectorAll('button, a, input, ...').length` |
| | F2 | event handler presence (onclick/onchange + registered listeners) | DOM audit + monkey-patch check |
| | F3 | focusable element count delta | tab sequence enumeration |

**Per operator output**: a 12-dim numeric vector — this is the **objective**
signature in paper table.

### 2.5 Agent behavioral signature (3 architectures)

Measured as **drop vs base**, per operator, per agent, averaged over tasks.

| Agent | Perception input | Model | Role |
|---|---|---|---|
| **Text-only** | DOM + a11y tree serialized as text | Claude Sonnet 4 | primary (most sensitive to semantic) |
| **SoM (Set-of-Mark)** | Screenshot + DOM annotations | Claude Sonnet 4 | hybrid (mixed modality) |
| **CUA (Coordinate)** | Screenshot only + coordinate actions | Claude Sonnet 4 | vision-only (most sensitive to visual) |

**Per operator output**: 3-dim behavioral drop vector
(text-drop, SoM-drop, CUA-drop).

### 2.6 Signature alignment (paper core contribution)

For each operator, compute alignment between:
- DOM signature vector (12-dim, objective)
- Behavioral signature vector (3-dim across agents)

**Expected alignments**:
- DOM signature with A1-A3 ≫ V1-V3, F1-F3 → **agent signature should show text-drop ≫ CUA-drop** → semantic-dominant operator
- DOM signature with V1-V3 ≫ A1-A3, F1-F3 → **agent signature should show CUA-drop ≫ text-drop** → visual-dominant operator
- DOM signature with F2 drop + F3 drop + rest unchanged → **agent signature should show all 3 agents drop similarly** → functional-dominant operator
- **Misalignment** — where DOM suggests one layer but behavior shows another — is itself a finding (agents don't map 1:1 to DOM changes; their observation modality filters)

## §11. Known limitations from reviewer audit (non-blocking for Mode A)

2026-04-28 ran 3 parallel audit sub-agents (code-review, production-
readiness, reviewer-#2) against today's A.4/A.5 deliverables. 4/4

**2026-04-28 PM session 2**: ran 3 additional parallel review agents

issues. **All 12 resolved** in commits 7e6f94c, f1b2cdd, 16072ea.
See §9 "All reviewer audit items resolved" table for full list.

The remaining items below are from the original AM audit and are
addressed in priority order after Mode A data lands.

### Parity + verification

- **Parity test is JSDOM-only.** Composite parity (§9.2 in
  `docs/amt-operator-spec.md`) compares change *counts* in JSDOM, not
  byte-identical DOM under Chromium. L5 (closed Shadow DOM) degrades
  silently under JSDOM. Strengthening path: add a Chromium-backed
  parity test producing normalized DOM hash per operator.
- **Per-operator parity is weaker than paper claim.** The test asserts
  `sum(countByType) equal` per family — per-operator counts could drift
  in compensating directions without failing. Strengthening path: break
  down parity per operator ID.
- **Double-run parity not tested.** L12's sentinel makes it non-
  equivalent under Plan D re-injection cycles vs. legacy apply-low.js.
  Strengthening path: add a "run operator twice, compare DOMs" test.

### 12-dim audit signature

- **V2 bbox is index-matched**; any operator that changes interactive
  count (L3, L5, L11, ML1) misaligns the pairing. Strengthening path:
  match bboxes by stable key (tag+nearest-centroid or injected audit-id).
- **A1/A2 uses multiset symmetric diff**, which double-counts every
  role/name change (remove + add). Paper tables need divide-by-2 or
  rename to "bag distance" for clarity.
- **F1 for L11 is tautological** — "interactive" selector excludes
  `<span>` so F1 is mechanically -N for N `<a>` elements. Paper must
  frame as measurement-apparatus limitation, not finding.
- **H4 A1=0 is tautological** — Chromium auto-maps `<nav>` to
  role=navigation, so *adding* `role="navigation"` doesn't change the
  AX tree. Paper must distinguish "DOM attribute set" (D2) from
  "computed AX tree changed" (A1).
- **Static-fixture validation only.** A.5 was smoke-tested on an
  85-line hand-written HTML. Framework interactions (KnockoutJS, Vue),
  Shadow DOM, iframes, delayed AX-tree population are all untested.
  **Required before any paper claim**: live-WebArena A.5 run with
  BEFORE-BEFORE variance floor documented.

### Framework-level

- **Composition canonical order H→ML→L** makes cross-family additivity
  tests measurement-asymmetric (e.g. H5c→L11 erases H5c's work; reverse
  does not). Paper's compositional study must run both orders or restrict
  additivity claims to commutative pairs only.
- **Signature alignment is not pre-registered.** As currently stated,
  any observed correlation (or anti-correlation) between DOM sig and
  behavioural sig is publishable as "alignment" or "informative
  misalignment" — unfalsifiable. Required before B.1: commit a
  prediction rule in §3 of the paper (e.g. "semantic-dominant operator
  IFF A1+A2 ≥ threshold AND V1 ≥ 0.99 AND |F*| ≤ ε").
- **Single-snapshot vs multi-step agent mismatch.** Audit measures
  t=0+quietMs; agents execute 5–30 steps. Paper positioning must
  acknowledge the audit is a t=0 DOM state, not an average over the
  agent's lived experience.

### Engineering

- **L11 onclick escape only handles single-quote.** href values
  containing `"`, `\`, or newline would break. Low severity for
  WebArena URLs today but worth addressing before any paper-cited
  operator behaviour.
- **H2 uses `tabindex="1"`** (positive tabindex = F44 violation).
  An "enhancement" operator that injects an F44 violation is
  paper-embarrassing. Trivial fix: change to `"0"`.
- **ML1 misses `role="button"`** — current ML1 swaps empty-text
  `<button>` with a bare `<div>`, not a `<div role="button">`. That
  makes it total non-compliance rather than pseudo-compliance, which is
  the ML family's whole point. Note: same bug exists in the legacy
  `apply-medium-low.js`; fixing ML1 without also fixing legacy breaks
  parity and invalidates composite-v1 data for ML1.
- **H5a alt heuristic doesn't strip query/fragment** from img src.
  Produces alt text like "logo png?v=42". Trivial fix.
- **CDP `Accessibility.enable` never explicitly disabled.** Not a leak
  today because each operator uses a fresh context, but a long-running
  audit with context reuse (H2 optimization) would need disable pairs.

### Performance / scale (surfaces at batch scale)

- **Audit opens 26 contexts per URL.** Docstring in audit-operator.ts
  claimed a shared BEFORE optimization; actual implementation doesn't
  share. At batch scale (13 URLs × 3 reps × 26 ops = 1,014 audits),
  this is ~10h on live WebArena vs ~2.7h with shared BEFORE. Fix path
  exists but is a Mode A–nice-to-have, not a blocker.
- **Screenshot PNGs never cleaned from `tmpdir`** — resolved as side
  effect of C4/H1 (now go to run-dir which is cleaned if needed), but
  the legacy `--output` mode still leaks PNGs to a sibling tmpdir.

1. **Docker state contamination.** Code inspection of
   `src/runner/scheduler.ts::executeExperiment` confirmed the

   the Magento / Postmill / GitLab databases. 73 of the 109 shard-B

   mutation). Under the planned Stage 3 matrix these 73 tasks would
   receive 26 × 3 × 2 = 156 writes each into a shared container. Same
   failure mode that forced Mode A post-hoc GT corrections on
   tasks 41/198/293.
2. **Trivial-ref false positive.** gitlab:306 (`must_include=['0']`,
   intent "How many commits did Anthony make between 08/2022-09/2022?")
   passed strict 3/3 gate because two of three reps emitted long
   confused prose that substring-matched `'0'` via the year "2023".
   Under manipulation, confused agent outputs will substring-match
   short tokens more often than clean outputs, asymmetrically
   under-counting drops on the 15 shard-B tasks with all-trivial
