## Detailed Figure Descriptions

### F1: AMT Framework Overview (GPT Image 2)

**What it shows**: The full experimental pipeline in one glance.
- Top: 26 operators organized in 3 families (L/ML/H)
- Middle: Dual measurement (DOM 12-dim + Behavioral 3-agent)
- Bottom: Signature alignment 2×2 matrix with results

**Key message**: "We have a systematic taxonomy, we measure it two ways,
and we cross-reference to find alignment/misalignment."

### F2: Injection Pipeline & Causal Decomposition (GPT Image 2)

**What it shows**: Two panels in one figure.
- Left: Plan D injection mechanism (how operators modify the DOM)
- Right: Three-agent causal decomposition (why we need 3 agents)

**Key message**: "The injection is invisible to the user (SSIM≈1.0),
and the three agents let us decompose effects into semantic vs functional."

### F3: Web Page Variant Example (GPT Image 2)

**What it shows**: A concrete e-commerce page under Base vs Low vs High.
- Three browser windows side by side
- Visually identical (Low looks same as Base)
- Red/green annotations showing DOM differences

**Key message**: "The page LOOKS the same but the DOM is devastated.
This is what the agent 'sees' differently."

### F4: Behavioral Drop Bar Chart (matplotlib)

**What it shows**: 26 operators sorted by Claude text-only drop from H-baseline.
- Horizontal bars, colored by family (L=red, ML=orange, H=green)
- Llama 4 drops as overlay markers (diamonds)
- H-baseline dashed line at 93.8%
- Key operators labeled: L1 (−40pp), L5 (−22pp), L12 (−14pp)

**Key message**: "Only 2-3 operators are truly destructive individually.
Most operators are below detection threshold."

### F5: DOM Signature Heatmap (matplotlib)

**What it shows**: 26 operators × 8 key DOM dimensions.
- Rows sorted by behavioral drop (same order as F4)
- Columns: D1, A1, A2, A3, V1, F1, F2, F3
- Color: diverging RdBu (red=high change, blue=low/negative)
- Normalize per column for visual comparability

**Key message**: "L11 has massive DOM changes across all dimensions.
L1 has modest changes. Yet L1 is more destructive."

### F6: Signature Alignment Scatter (matplotlib) — CORE FIGURE

**What it shows**: DOM magnitude (x) vs behavioral drop (y) for all 26 operators.
- X-axis: composite DOM change score (sum of normalized dims)
- Y-axis: Claude text-only drop (pp)
- Points colored by alignment category
- Key operators annotated: L1 (low DOM, high drop), L11 (high DOM, low drop), L5 (high both)
- Quadrant lines at thresholds

**Key message**: "DOM change magnitude is ANTI-CORRELATED with behavioral
impact. The misalignments (L1, L11) are the paper's core finding."

### F7: Cross-Model Comparison (matplotlib)

**What it shows**: Claude vs Llama 4 text-only drops for top-10 operators.
- Cleveland dot plot (connected dots)
- Left dot = Claude, right dot = Llama 4
- Highlight L11 gap: Claude +1.5pp vs Llama +14.6pp
- Highlight L1/L5 agreement: both models show large drops

**Key message**: "The a11y effect generalizes across models, but
adaptation capacity differs (L11 adaptive recovery gap)."

### F8: Compositional Scatter (matplotlib)

**What it shows**: Expected vs observed drop for 28 pairwise combinations.
- X = sum of individual drops (expected if additive)
- Y = observed pair drop
- Diagonal = perfect additivity
- Points above diagonal = super-additive (red)
- Points on diagonal = additive (gray)
- Points below diagonal = sub-additive (blue)
- Key pairs labeled: L6+L11 (+24pp interaction), L1+L5 (−17pp sub-additive)

**Key message**: "50% of pairs are super-additive. Operators amplify
each other. The composite low variant's −70pp drop is explained by
interaction chains (L1+L11+L6)."

### F9: Per-Task × Operator Heatmap (matplotlib, supplementary)

**What it shows**: Full 13 tasks × 26 operators success rate matrix.
- Cell color: 0% (dark red) to 100% (dark green)
- Cell text: percentage
- Row labels: task ID + app name
- Column labels: operator ID
- Highlight: task 67 (most variable), task 132/188 (all 100%)

**Key message**: "Task sensitivity varies enormously. Some tasks are
operator-immune (132, 188), others are highly discriminating (67, 4)."

**Left panel** — Stage 3 breadth pipeline (top → bottom):

**Right panel** — Mode A (N=13) depth set retrospective check:
- Display all 13 tasks as a small inline table

**Key message**: The 684 → N drop looks drastic in aggregate, but every

## GPT Image 2 Best Practices (from OpenAI official guide, 2026-04-21)

1. **5-element formula**: Subject → Style → Composition → Details → Constraints
2. **Explicit layout**: Specify positions ("top-left", "3 columns", "55%/45% split")
3. **Text rendering**: GPT Image 2 handles text well — include exact labels
4. **Color as hex**: `#C0392B` not "red" for precision
5. **Aspect ratio**: Specify explicitly ("16:9 landscape, 2560×1440px")
6. **Style anchoring**: "academic paper figure", "CHI/Nature journal style"
7. **Negative constraints**: "No shadows, no gradients, no 3D, no decorative elements"
8. **Thinking mode**: Use for complex multi-panel layouts (paid subscribers)
9. **Iterate**: "Keep everything but fix X" for targeted corrections
10. **Resolution**: Request 2K+ (2560×1440) for print quality

**Key insight from testing**: GPT Image 2 reasons about prompts before
generating. It understands spatial relationships, can render accurate text,
and maintains consistency across edits. Structure your prompt as a
specification, not a description.
