### Step 2: Compute per-capture SSIM

Per URL, diff every variant screenshot against the URL's `base.png`. Also
compute `base2 vs base` for baseline-noise floor. You probably want per-
URL-per-variant rows with `{slug, app, url, variant, visits, ssim, phash_dist, wcag_delta}`.

Key metrics to compute per variant pair `(base, X)`:
- **SSIM** (`skimage.metrics.structural_similarity`, `channel_axis=2`) — primary
- **pHash** (imagehash library, hamming distance) — secondary, catches layout shifts SSIM misses
- **WCAG contrast delta** — optional, for operators that might change foreground/background

Reference implementation: `analysis/visual_equivalence_analysis.py` (from Phase 7,
reusable for the aggregation logic).

### Step 3: Aggregate per operator

Target CSV: `results/stage3/visual-equiv/ssim-per-operator.csv` with columns:
```
operator, n_urls, ssim_mean, ssim_p50, ssim_p10, ssim_p90, ssim_std,
  phash_mean_dist, phash_p50
```

Paper claim to test: **"Semantic operators (L2, L7, L10, L12, L13, all ML,
all H) preserve visual rendering (SSIM ≥ 0.99), while structural operators
(L5 Shadow DOM, L6 heading→div, L9 table elements) produce measurable
visual change (SSIM < 0.97)."**

Early 5-URL spot-check (2026-05-10, before full run completed) strongly
supported this — see `results/stage3/visual-equiv/` trailer notes below
and chat log.
