- `ps -ef | grep run-pilot3 | wc -l` returns 0 AND

## Known gotchas (read before touching anything)

### 1. Model name is Sonnet 4, NOT 3.5

LiteLLM alias `claude-sonnet` routes to
`us.anthropic.claude-sonnet-4-20250514-v1:0` (Claude Sonnet 4,
released May 2025). Old docs incorrectly called it "Sonnet 3.5" — all
corrected in commit `88cccca` (2026-05-05).

If you see "Claude Sonnet 3.5" anywhere now, it's either (a) in
`figures/archive/` (frozen, leave alone) or (b) a capability-floor note
about Anthropic Computer Use tool (literally correct — Anthropic's
docs say CU requires 3.5 v2+).

### 2. ada credentials expire in 1h

Every SSM call needs fresh creds. If you see
`ExpiredTokenException`, run `ada credentials update` first.

### 3. SSM wraps processes in a process group that dies with SSM

For ANY long-running process launched via SSM (LiteLLM, smoker), you
MUST use `nohup setsid bash -c '...' </dev/null > log 2>&1 & disown`.
Plain `nohup ... &` will be killed when SSM exits. See commit `7e35934`
for fix rationale.

### 4. `python` symlink (AL2023 gotcha)

AL2023 has no `/usr/bin/python` by default. Runner spawns `python`
(not `python3.11`) so we need:

```bash
sudo ln -sf /usr/bin/python3.11 /usr/local/bin/python
```

Do NOT overwrite `/usr/bin/python3` — that breaks `dnf`. Use
`/usr/local/bin/` instead. This is baked into
`scripts/ssm/ssm-bootstrap-smoker.json`.

### 5. Python Playwright has its own chromium version

Node Playwright's chromium (via `npx playwright install`) is
different from Python Playwright's (chromium-1117 vs 1217 in
practice). Bridge uses Python → needs `python -m playwright install
chromium` separately. Baked into bootstrap.

### 6. Magento admin grids exceed Claude's context window

Already fixed (commit `7e9bb1c`) by truncating a11y tree to 40K chars
(head 30K + tail 5K). Verify this doesn't silently break Stage 3 by
checking `ContextWindowExceededError` count in the first ~50 cases.

### 7. BrowserGym serialization divergence (paper-relevant, not bug)

`aria-hidden="true"` shows as `hidden=True` in BrowserGym's
serialization but elements are still clickable. This is a known
platform-level finding documented in project-phases.md Phase 2. NOT a
bug to fix — it's a published limitation.

### 8. Stage 4b (trace-URL SSIM audit) is the CUA replacement

The CUA agent is NO LONGER the visual control. Don't add CUA to
Stage 3 manipulation just because "that's what we did before". The
Stage 4b pixel-level audit is the direct visual measurement. See
submission roadmap.

### 9. Don't git push to main/master lightly

Per amazon-builder-git rules, never force push; never rewrite pushed
history; use `git push` (tracked branch) only. All our commits today
pushed via `git push origin master` — that's fine.

### 10. Answer-consistency check was REMOVED from the gate

Earlier draft of the filter required all successful reps to produce
the same normalized final answer. We **dropped** this 2026-05-06 —
BrowserGym's evaluator is authoritative; our literal-string check was
false-rejecting paraphrase variation. Don't re-add it.
